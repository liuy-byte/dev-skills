# 连载系列写作

写微信公众号的连载教程系列（「N 天」「DayN」型，如「FastAPI 14 天」「Docker 14 天」）。覆盖三件事：① 给主题规划 N 天大纲；② 逐篇生成符合连载格式的文章；③ 为已写好的单篇补连载格式（命名/承接/总结表/预告）。

> **通用写作规范**（标题钩子、语言风格、禁忌清单、图床 URL、发布前自检、发布闭环命令）一律沿用同目录 `viral-article.md`，本文档只补充**连载特有**的规划与格式规则。
> 单篇非连载文章见 `viral-article.md`。

## 三种用法

| 用法 | 触发 | 产出 |
|------|------|------|
| **规划** | "做个 X 的 7 天连载" | N 天大纲（每天主题 + 标题钩子） |
| **逐篇生成** | "写 DayN" / 按大纲继续 | 一篇符合连载格式的完整文章 |
| **补格式** | "把这几篇做成连载" | 给已有单篇补命名、承接句、总结表、预告 |

---

## 一、命名规范

沿用现有系列的 `-` 分隔、Day 编号居中风格：

```
YYYYMMDD-<系列名>-<N>天-Day<n>-<标题>.md
```

- 普通篇：`20260418-Docker-14天-Day5-多阶段构建和构建缓存-让镜像更小更快.md`
- 收尾篇：`20260418-Docker-14天-Day14-全景知识图谱-面试通关指南.md`

**系列名内部若含空格，统一转为 `-`**（如 `Claude Code` → `Claude-Code`）：
`20260604-Claude-Code-7天-Day1-5分钟跑起第一个会话.md`

保存在 `articles/` 子模块根目录。

---

## 二、规划阶段（给主题 → N 天大纲）

输入「主题 + 天数」，产出 N 天大纲。规则：

- **Day1 锁"5分钟"钩子**：「5分钟搞懂…」「5分钟跑起…」。开篇**不列大纲**，直接进正文。
- **中间篇用代入感钩子**：「像追剧一样…」「像搭积木一样…」「一次搞懂…」「告别手写…」，每篇聚焦**一个核心概念**。
- **收尾篇固定为"全景知识图谱-面试通关指南"**，不叫 DayN+1。
- 若系列配套代码仓库（如 `fastapi-14days`、`langgraph-7days`），收尾篇用**后台回复关键字取代码**驱动，不在正文贴仓库链接。

大纲输出示例：

```
Day1 ｜5分钟搞懂 X 核心概念
Day2 ｜像追剧一样看 Y 变化
...
DayN ｜全景知识图谱-面试通关指南（后台回复「关键字」取代码）
```

## 三、逐篇格式

### 开篇（每篇都有）

```markdown
# <系列名> N 天系列 Day n｜<核心卖点>

> [痛点，1-2句]
> [价值承诺，1-2句]

---

## 一、...
```

### Day 之间的承接

- **开头承接**（可选，Day2 起）：
  - `昨天学了 X，但 Y 怎么…？`
  - `前一篇讲了概念，这篇进入实操。`
- **篇末三件套**（普通篇必有）：

```markdown
---

## Day n 总结

| 概念 | 一句话说明 |
|------|-----------|
| ... | ... |

---

## 下一篇预告

Day n+1 会讲 ...

---

## 相关链接

微信内长按复制链接到浏览器打开。

​```
官方文档：https://...
GitHub：https://...
​```
```

### 篇幅控制

连载单篇比独立爆款文**更短、更聚焦**：3–8 个汉字编号章节，每个概念配**一个核心代码示例**即可，不堆砌。

---

## 四、收尾篇（DayN：全景知识图谱-面试通关指南）

固定结构：

1. **全景图谱**（ASCII 框图，把整个系列的模块/命令/概念串成一张图）
2. **核心速查表**（命令 / API / 概念对照表）
3. **高频面试题** `Q1…Qk`（问题 + 参考答案）
4. **后台关键字入口**（引导后台回复关键字领取资料）：
   - 有配套代码仓库（如 `fastapi-14days`）：领取完整项目源码
   - 无代码仓库的工具类连载（如 Claude Code）：领取配置模板 / 命令速查表 / 笔记

```markdown
## 📬 获取完整资料

回复关键字 **`<keyword>`** 到公众号后台，获取 <项目源码 / 配置模板与速查表>，包含全部 N 天内容。
```

---

## 五、发布

完成后走 `viral-article.md` 的**第八步发布前自检**与**第九步发布闭环**（articles → images → 父仓库 bump 的子母仓库提交命令，用户确认后执行）。连载额外检查：

- [ ] 命名 `YYYYMMDD-系列名-N天-DayN-标题.md` 规范
- [ ] 开篇钩子、承接句、Day N 总结表、下一篇预告齐全
- [ ] 收尾篇有全景图谱 + 面试题 + 后台关键字入口
- [ ] 与同系列前序篇的术语、风格一致
