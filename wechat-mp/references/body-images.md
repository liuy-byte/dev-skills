# 正文配图

为技术类公众号文章规划、生成、编辑和规整正文配图。目标是帮助读者理解内容，不是用装饰图填空。

## 工作流

1. 完整阅读文章，按章节列出配图计划：插入位置、图片作用、图片类型、来源或生成方式、文件名。
2. 用户明确数量时严格按数量生成；未指定时通常选 3–5 张，不要机械地每节一张。
3. 按下方素材优先级选择方案。能用真实截图或确定性图形表达时，不生成抽象概念图。
4. 生成或编辑后逐张目视检查，再运行校验脚本。
5. 将图片保存到 `images/<文章专属文件夹>/`，在正文对应段落后插入图床 raw URL。

配图计划使用简表即可：

| 文件名 | 插入位置 | 作用 | 类型 | 来源/生成方式 |
|---|---|---|---|---|
| `opencode-tui.png` | “进入 TUI”段落后 | 展示真实操作界面 | 产品截图 | 官方截图裁切 |

## 素材优先级

从上到下选择，只有上一级不能满足文章表达时才降级：

1. 用户提供的截图或已有项目图片；
2. 产品真实界面、官方文档或官方仓库中的截图；
3. 用 HTML/CSS、SVG、Mermaid 或绘图工具生成的确定性流程图、架构图、对比图；
4. AI 生成的克制型概念插图。

引用外部图片前确认使用范围和来源；来源不明确时不要冒充官方截图。修改已有图片时先查看原图，默认保留原文件并输出新版本；用户明确要求替换时才覆盖。

## 按文章内容选择图片类型

| 内容 | 优先图片 |
|---|---|
| 安装、配置、命令执行 | 真实终端或产品截图 |
| 工作流、调用顺序、迁移步骤 | 规整流程图 |
| 模块关系、系统组成 | 架构图 |
| 方案差异、选型 | 对比卡片或表格图 |
| 抽象观点、行业背景 | 克制的概念插图 |

技术教程默认避免漂浮的机器人、宇宙背景、发光人物、夸张 3D 图标和与正文无关的装饰。每张图只表达一个核心信息。

## 默认视觉规格

同一篇文章的正文图默认统一为：

- 2160×1440，3:2；
- PNG；照片类素材可以使用高质量 JPEG；
- 相同画布、边距、圆角、背景亮度和视觉密度；
- 主体与画布边缘保留安全距离；
- 禁止拉伸原图；比例不一致时使用裁切或留白画布；
- 长页面截图可以例外，但同类长图仍应保持相同宽度。

用户或项目已有明确尺寸规范时优先遵循项目规范。

## 生成约束

生成图片前把文章中的真实名词、界面、产品关系和必须出现的文字列为硬约束。提示词至少说明：

- 图片在文章中解释什么；
- 画面主体和信息层级；
- 与同篇其他图片共享的比例、颜色和留白；
- 必须保留的准确文字；
- 不允许出现的无关元素和虚构 UI。

技术正文图不要默认添加大标题或口号。需要文字时优先使用确定性排版；AI 生图只负责背景或无文字插图。

## 截图和文字编辑

终端、代码、模型名、命令和产品 UI 属于文字敏感素材：

1. 优先重新截取真实界面，或只对目标区域做局部修改；
2. 不要为了改一个词重新生成整张截图；
3. 保持原图尺寸、裁切、字体、颜色、间距和其他文本不变；
4. 修改后逐字检查目标文字，同时确认其他文字没有漂移、缺失或被改写；
5. 生成式编辑出现界面变形或文字失真时，放弃该结果，回到原图，改用真实截图或确定性排版。

不得虚构命令执行结果、模型能力、价格、性能数据或不存在的产品界面。

## 插入正文

图片紧跟它解释的段落或步骤，避免集中堆在章节开头或文末。Markdown 使用：

```markdown
![准确描述图片内容的说明](<图床 raw URL>)
```

raw URL 规则沿用 `viral-article.md`：URL 中不包含本地 `images/` 层级。

## 自动校验

对本次正文图片显式传入文件路径，避免把同目录的封面一起检查：

```bash
node /abs/path/to/wechat-mp/scripts/body-images/validate.mjs \
  /abs/path/to/images/article/image-1.png \
  /abs/path/to/images/article/image-2.png \
  --width 2160 --height 1440 --count 2
```

脚本检查：

- 文件是否存在且为 PNG/JPEG；
- 像素尺寸是否符合要求；
- 数量是否正确；
- 文件是否为空或异常大；
- 是否存在内容完全相同的重复图。

脚本通过不代表视觉合格。完成前仍需目视检查图片与正文的相关性、文字准确性、裁切、留白和整套一致性。

## 完成前检查

- [ ] 数量与用户要求一致
- [ ] 每张图都有明确的信息作用和插入位置
- [ ] 优先使用了真实截图或确定性图形
- [ ] 没有无关、浮夸或虚构的视觉元素
- [ ] 同篇图片尺寸、比例、边距和风格一致
- [ ] UI、命令、模型名和代码文字准确
- [ ] 没有拉伸、误裁切、模糊或重复图片
- [ ] 文件位于文章专属目录，正文使用正确 raw URL
